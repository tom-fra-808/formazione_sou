COS'E'
Un git conflict è un errore di git in cui il programma non sa decidere quali file o parti di file tenere quando si va a fare un merge di due branch.

COME SI GENERA
Un git conflict può essere generato quando, su due branch diversi, vengono modificati contemporaneamente degli stessi file o rimossi oppure quando vengono modificate linee di uno stesso file. Il conflitto vero e proprio si verifica quando si prova a fare merge.

COME RISOLVERE
Per risolvere l'errore si controllano i file in conflitto. Andando a vedere per esempio un file che è stato modificato in modi diversi in due branch distinte, questo apparirà diviso in due parti: una corrispondente al branch main ed una al secondo branch.
			<<<<<<< HEAD
			Mia distro preferita: ubuntu24
			=======
			Mia distro preferita: rocky8
			>>>>>>> distro-2

Dopo aver modificato il file e aver deciso se tenere una, l'altra o entrambe le varianti, si può riaggiungere quest'ultimo a git con 'git add "file"'. Dopodiché si può continuare il merge.
